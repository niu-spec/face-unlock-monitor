# 2\. **产品概述**

## 2\.1 **项目背景与意义**

当前，智能家居安防需求日益增长，用户希望通过家庭摄像头实时监控家中情况，保障家人安全。传统监控系统仅具备录像功能，无法实现智能分析和主动告警。随着计算机视觉和深度学习技术的发展，基于AI的智能监控系统能够自动识别人脸、检测异常行为、识别危险情况，大幅提升家庭安全防护能力。

本项目旨在构建一套居家智能摄像头监控系统，实现人脸识别（区分家人与陌生人）、危险区域管控（如厨房禁止小孩进入）、异常事件检测（积水、着火、摔倒）等核心功能，为用户提供全方位的智能安防解决方案。

## 2\.2 **项目目标（项目主要工作内容）**

**1\.人脸识别与人员管理**：基于dlib实现128维人脸特征提取，支持家人注册、实时识别、陌生人检测告警，并提供实时人数统计功能。

**2\.危险区域智能管控**：支持Canvas绘制多边形危险区域，设置禁区规则（如禁止小孩进入厨房），实现区域闯入检测与告警。

**3\.异常事件自动检测**：实现积水、着火、摔倒三种异常检测，通过图像分析自动识别危险情况并触发告警。

## 2\.3 技术方案

系统采用前后端分离架构：Vue3 前端负责监控展示与交互，Django \+ DRF 后端承载业务 API 与数据持久化，AI 分析模块（OpenCV / dlib）与视频拉流同进程集成，通过 Nginx\-RTMP 接收摄像头推流，经 MJPEG 向前端输出带标注的实时画面。

|层次|技术选型|说明|
|---|---|---|
|前端|Vue3 \+ Vite \+ Element Plus|监控大屏、Canvas 区域画框、告警列表、家人录入|
|业务后端|Django \+ DRF \+ JWT|用户认证、区域/告警/事件 API，Swagger 文档|
|人脸识别|dlib \+ face\_recognition|128 维特征编码，家人库比对与陌生人检测|
|目标检测|OpenCV HOG|行人检测，配合禁区多边形判断闯入|
|异常检测|OpenCV 颜色/形态分析|积水（HSV）、着火（高亮区域）、摔倒（高宽比）|
|数据库|MySQL|用户、家庭成员、危险区域、告警与事件日志|
|流媒体|Nginx\-RTMP|RTMP 推流（:9090）→ 后端拉流 → MJPEG 展示|

|用例ID|用例名称|参与者|前置条件|后置条件|

**业务流程**：

后端调用face\_service提取128维人脸特征编码，存入registered\_faces\.json和MySQL

**功能描述**：基于图像分析实现积水、着火、摔倒三种异常检测。

|用例ID|用例名称|参与者|前置条件|后置条件|
|---|---|---|---|---|

|UC\-009|着火检测|系统|高亮红/黄区域阈值判断|FIRE|
|UC\-010|摔倒检测|系统|人体框高宽比低于阈值|FALL|

**业务流程**：

从视频流中提取帧图像

积水检测：转换HSV色彩空间，检测画面下部蓝色/灰色大面积区域

着火检测：检测高亮度红/黄区域，超过面积阈值则判定为着火

摔倒检测：通过HOG检测人体，计算人体框高宽比，低于阈值判定为摔倒

任一异常检测触发，调用alert\_service创建告警

告警包含：异常类型、检测时间、置信度、截图



### 3\.2\.4** 视频监控模块**

**功能描述**：基于 Nginx\-RTMP 实现视频流推流与拉流，支持多路摄像头实时画面展示、视角切换与历史回放。

|用例ID|用例名称|参与者|前置条件|后置条件|
|---|---|---|---|---|
|UC\-011|实时视频查看|管理员|摄像头已配置并推流|前端显示 MJPEG 实时画面|
|UC\-012|多摄像头切换|管理员|存在多路摄像头配置|切换至指定摄像头画面 |
|UC\-013|视频流拉流|系统|RTMP 服务正常运行|输出 MJPEG 帧流|
|UC\-014|历史回放|管理员|告警/事件已关联截图|展示历史截图或片段 |

**业务流程**：

管理员通过前端 HomeMonitor\.vue 进入监控页面，系统调用 /video\_feed/\{id\} 接口获取视频流。

后端 video\.py 通过 ffmpeg 从 Nginx\-RTMP 拉取 RTMP 流，提取单帧图像并转换为 MJPEG 格式。

前端通过 \<img\> 标签接收 MJPEG 流，实现无插件实时画面展示。

管理员点击摄像头切换按钮（客厅/厨房），前端更换 video\_feed 的 id 参数，重新建立拉流连接。

系统支持多路摄像头同时在线，通过 gen\_frames 框架为每帧预留 AI 画框钩子。

告警触发时，snapshot\_service 自动截取当前帧并保存至 /static/snapshots/，供后续回放查询。



### 3\.2\.5** 告警中心模块**

**功能描述**：统一接收人脸识别、区域管控、异常检测三大模块触发的告警，提供告警展示、筛选、处置、日志查询与截图回放功能。

|用例ID|用例名称|参与者|前置条件|后置条件|
|---|---|---|---|---|
|UC\-015|查看告警列表|管理员|系统已产生告警记录|展示分页告警列表|
|UC\-016|按类型筛选告警|管理员|告警列表已加载|显示筛选后结果|
|UC\-017|处置告警|管理员|告警状态为 pending|告警状态更新为 handled|
|UC\-018|告警截图回放|管理员|告警已关联 snapshot\_path|展示告警触发时截图|
|UC\-019|查看告警日志|管理员|事件已记录|展示时间线事件记录|

**业务流程**：

人脸识别模块触发 FACE\_UNKNOWN、区域管控模块触发 ZONE\_INTRUSION、异常检测模块触发 FLOOD/FIRE/FALL 时，统一调用 alert\_service\.create\_alert\(\)。

alert\_service 将告警信息（类型、级别、描述、截图路径、状态）写入 MySQL alert 表，并通过 WebSocket 推送至前端。

前端 AlertCenter\.vue 接收推送，实时弹窗并播放告警提示音，同时更新告警列表。

管理员进入告警中心，可按类型（FACE\_UNKNOWN/ZONE\_INTRUSION/FLOOD/FIRE/FALL）、状态（pending/handled/ignored）、时间范围筛选告警。

管理员点击"处置"按钮，调用 PUT /api/alerts/\{id\}/handle，后端更新 alert 状态为 handled 并记录 handle\_time。

管理员点击"回放"按钮，前端根据 snapshot\_path 加载告警触发时的截图，展示事件现场。

EventLog\.vue 展示告警与识别事件的时间线，支持按日期查询历史记录。



### 3\.2\.6** 用户管理模块**

**功能描述**：提供基于 JWT 的用户认证与授权，支持管理员登录注册、家人信息管理、角色权限控制及系统基础设置。

|**表格****用例 ID**|**用例名称**|**参与者**|**前置条件**|**后置条件**|
|---|---|---|---|---|
|UC\-020|用户登录|管理员|账号已注册|签发 JWT Token，进入系统|
|UC\-021|用户注册|管理员|手机号未注册|创建新用户账号|
|UC\-022|家人信息管理|管理员|已登录系统|新增/修改/删除家人信息|
|UC\-023|权限控制|系统|用户已登录|校验操作权限|
|UC\-024|系统设置|管理员|已登录且为 superuser|更新系统配置|

**业务流程**：

管理员在前端 Login\.vue 输入手机号和密码，前端通过 Axios 发送 POST /api/auth/login 请求。

后端 auth\.py 校验密码（bcrypt 比对），验证通过后调用 JWTManager\.encode\(\) 生成 Token，返回给前端。

前端将 Token 存入 localStorage，后续请求通过 Axios 拦截器自动注入 Authorization 头部。

后端通过 JWT 中间件校验 Token 有效性，过期或无效时返回 401，前端跳转登录页。

管理员进入 FamilyRegister\.vue 录入家人信息（姓名、角色 adult/child），系统调用家人注册 API 完成人脸录入与数据持久化。

权限控制通过 role 字段（admin/member）实现，非 admin 用户无法进入系统设置与区域编辑页面。

系统设置页面支持配置网络参数、摄像头 RTSP 地址、告警阈值等，仅 superuser 可访问。
