# 实时视频分析监测系统 — 总体架构说明

> **项目名称**：home-camera-monitor  
> **应用场景**：居家智能摄像头监控  
> **版本**：v1.4  
> **编制日期**：2026-07-10  
> **技术方案**：Vue3 + Django 4.2（业务 + AI 同进程）+ OpenCV/dlib + MediaMTX + MySQL

---

## 1. 文档目的

本文档描述居家摄像头监控系统的总体架构、技术选型、模块划分、数据流与部署方案，供立项评审、中期检查与结题验收使用。

## 2. 项目背景与目标

### 2.1 背景

现代家庭对居家安全与成员看护的需求日益增长。本系统基于家庭摄像头实时视频流，结合人脸识别与计算机视觉分析，实现对家庭成员的识别、在家人数统计、陌生人检测，以及厨房等危险区域的闯入告警，并对积水、着火、人员摔倒等异常情况进行自动识别与告警。

### 2.2 建设目标

| 目标 | 说明 |
|------|------|
| 家庭成员识别 | 注册家庭成员人脸，实时识别并标注身份 |
| 人员统计 | 统计当前画面/家中人数，区分熟人与陌生人 |
| 陌生人告警 | 未注册人脸出现即触发告警 |
| 危险区域 | 可配置禁区（如厨房禁止小孩进入），闯入即告警 |
| 异常检测 | 积水、着火、人员摔倒等紧急情况识别（≥2 种） |
| 告警管理 | 统一告警展示、处置、日志与事件回放 |

### 3.2 架构设计原则

| 原则 | 说明 |
|------|------|
| Django 单体集成 | 业务 API 与 AI 模块均为 `backend/apps/` 下的 Django App，同进程运行 |
| 告警服务化 | AI 模块通过 `apps.alerts.services.create_alert()` 写库（亦可 HTTP POST `/api/alerts/`） |
| 推拉流分离 | MediaMTX 接收 RTMP 推流并转发 RTSP，Django OpenCV 拉流分析 |
| 多家庭隔离 | 用户可加入多个家庭，前端携带 `X-Active-Household-Id` 请求头 |
| 前后端分离 | 前端通过 REST API 访问业务层，通过 MJPEG 展示实时画面 |

## 4. 技术选型

| 层次 | 技术 | 说明 |
|------|------|------|
| 前端 | Vue3 + Vite + Element Plus + Vue Router | 监控大屏、区域画框、告警列表 |
| 业务后端 | Django 4.2 + DRF + SimpleJWT | 认证、CRUD、Swagger 文档 |
| API 文档 | drf-yasg | `/api/docs/` 自动生成 |
| AI 运行环境 | conda `home-camera`（Python 3.10） | Windows 上 dlib 必须通过 conda 安装 |
| 人脸识别 | dlib + face_recognition | 128 维编码，家人库比对 |
| 行人检测 | YOLOv8n（优先）+ OpenCV HOG 降级 | 危险区域闯入 / 接近 / 逗留 |
| 异常检测 | OpenCV HSV / 形态分析 + 多信号融合 | 积水、着火、跌倒 |
| 数据库 | MySQL 8.x | 用户、家庭、区域、告警、事件 |
| 流媒体 | MediaMTX（Docker） | 替代原 Nginx-RTMP 方案，RTMP 入 / RTSP 出 |
| 部署 | gunicorn + Nginx 反代 | 生产环境 API 与静态资源 |

## 5. 推拉流方案

| 环节 | 协议 | 端口 | 说明 |
|------|------|------|------|
| 推流 | RTMP | 9090 | 摄像头/手机/OBS → MediaMTX |
| 中转 | RTSP | 8554 | MediaMTX 内部转发 |
| WebRTC 预览 | HTTP | 8889 | 浏览器低延迟预览（MediaMTX） |
| 拉流 | OpenCV `VideoCapture` | — | Django `video_stream` 读取 RTSP |
| 展示 | MJPEG | 8010（生产）/ 8000（本地） | `/video_feed/{id}` |

```
推流：rtmp://{IP}:9090/stream/1
拉流：rtsp://127.0.0.1:8554/stream/1
WebRTC：http://{IP}:8889/stream/1/
MJPEG：http://{IP}/video_feed/1  （生产经 Nginx；本地 Vite 代理 :5173）
```

### 流 ID 双轨约定

视频层（MediaMTX / OBS 推流码）与业务层（zones / alerts / events）使用不同 ID，前端 `constants/streams.js` 负责映射：

| 场景 | 视频流 ID | 业务 stream_id |
|------|-----------|----------------|
| 客厅 | `1` | `living_room` |
| 厨房 | `2` | `kitchen` |

### 视频处理流水线（已实现）

```
RTSP 帧 → 跳帧（FRAME_SKIP）
    → process_frame() 处理器链：
        ① face.process_frame()      — 人脸检测/识别/画框/人数统计
        ② detection.process_frame() — YOLO/HOG 行人 + 禁区/接近/逗留 + 积水/着火/跌倒
        ③ detection.draw_overlays() — 区域多边形
        ④ face.draw_face_boxes()    — 人脸框最终绘制
    → MJPEG 编码输出
    → create_alert() 写入告警表 + 事件快照
```

WebRTC 预览无内嵌 AI 框时，前端 `FaceOverlay.vue` 轮询 `/api/video/status` 在 Canvas 上叠加。

### 6.2 核心业务场景

| 场景 | 触发条件 | 系统行为 | 告警类型 |
|------|----------|----------|----------|
| 家人识别 | 已注册家庭成员出现 | 绿框 + 显示姓名 | — |
| 陌生人 | 未注册人脸 | 红框 + 告警 | `FACE_UNKNOWN` |
| 人数统计 | 画面中多张人脸 | 更新在家人数/陌生人数 | 前端轮询展示 |
| 小孩进厨房 | 角色=child 进入厨房禁区 | 告警 | `INTRUSION` |
| 距边缘过近 | 禁区外 safe_distance 内 | 告警 | `PROXIMITY` |
| 异常停留 | 禁区附近 dwell_time 超时 | 告警 | `LOITER` |
| 积水 | 画面低处大面积蓝色/反光区域 | 告警 | `WATER` |
| 着火 | 高亮/红色火焰颜色区域 | 告警 | `FIRE` |
| 摔倒 | 人体框高宽比异常（躺倒） | 告警 | `FALL` |

| created_at | DATETIME | 发生时间 |

## 10. 关键业务流程

### 10.1 人脸识别与人数统计

```
RTSP 视频帧 → video_stream.process_frame()
    → face.services.process_frame()
    → 检测所有人脸 → 与家人员比对
    → 统计：total / family / stranger
    → 更新内存快照 → GET /api/home/presence/ 可读
    → 陌生人 → create_alert(FACE_UNKNOWN)
    → MJPEG 标注框输出
```

### 10.2 危险区域（厨房禁小孩）

```
zone 配置：厨房多边形 + forbidden_roles=["child"]
    → detection 服务 HOG 检测行人
    → 结合 face 识别的 role + 行人进入多边形
    → create_alert(INTRUSION)
```

### 10.3 异常检测（积水/着火/跌倒）

```
每 N 帧在 process_frame() 链中执行：
    WATER  — 画面下方 HSV 蓝色/反光大面积 → create_alert(WATER)
    FIRE   — 高亮红/黄火焰颜色区域超阈值 → create_alert(FIRE)
    FALL   — 人体框高/宽比低于阈值（躺倒）→ create_alert(FALL)
```
