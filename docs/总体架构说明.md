# 实时视频分析监测系统 — 总体架构说明

> **项目名称**：home-camera-monitor  
> **应用场景**：居家智能摄像头监控  
> **版本**：v1.2  
> **编制日期**：2026-07-07  
> **技术方案**：Vue3 + Django（业务）+ MediaMTX + AI 服务（OpenCV/dlib）+ MySQL

---

## 1. 文档目的

本文档描述居家摄像头监控系统的总体架构、技术选型、模块划分、数据流与部署方案，供立项评审、中期检查与结题验收使用。

---

## 2. 项目背景与目标

### 2.1 背景

现代家庭对居家安全与成员看护的需求日益增长。本系统基于家庭摄像头实时视频流，结合人脸识别与计算机视觉分析，实现对家庭成员的识别、在家人数统计、陌生人检测，以及厨房等危险区域的闯入告警，并对积水、着火、人员摔倒等异常情况进行自动识别与告警。

### 2.2 建设目标

| 目标 | 说明 |
|------|------|
| 家庭成员识别 | 注册家庭成员人脸，实时识别并标注身份 |
| 人员统计 | 统计当前画面/家中人数，区分熟人与陌生人 |
| 陌生人告警 | 未注册人脸出现即触发告警 |
| 危险区域 | 可配置禁区（如厨房禁止小孩进入），闯入即告警 |
| 异常检测 | 积水、着火、人员摔倒等紧急情况识别（≥2 种） |
| 告警管理 | 统一告警展示、处置、日志与事件回放 |

### 3.2 架构设计原则

| 原则 | 说明 |
|------|------|
| 业务与 AI 分层 | Django 负责业务 API 与数据持久化；AI 模块独立开发，通过 HTTP 写入告警/事件 |
| 推拉流分离 | MediaMTX 接收 RTMP 推流，Django/OpenCV 通过 RTSP 拉流分析 |
| 场景驱动 | 功能围绕居家看护：识人、数人、禁区、异常 |
| 前后端分离 | 前端通过 REST API 访问业务层，通过 MJPEG 展示实时画面 |

### 3.3 架构演进说明

| 阶段 | 方案 | 状态 |
|------|------|------|
| v1.0 | Flask 单体（AI + 业务同进程） | 已废弃 |
| v1.2 | Django 业务层 + AI 服务分层 | **当前方案** |

业务 API 已从 Flask 迁移至 Django；视频拉流、人脸识别、异常检测模块在 `feature/django-video-stream`、`feature/face`、`feature/detection` 等分支并行开发，完成后通过 HTTP 接入 Django 告警/事件接口。

## 4. 技术选型

| 层次 | 技术 | 说明 |
|------|------|------|
| 前端 | Vue3 + Vite + Element Plus | 监控大屏、区域画框、告警列表 |

## 5. 推拉流方案

| 环节 | 协议 | 端口 | 说明 |
|------|------|------|------|
| 推流 | RTMP | 9090 | 摄像头/手机/OBS 推到 MediaMTX |
| 拉流 | RTSP/OpenCV | 8554 | 视频流后端内部读取 |
| 展示 | MJPEG | 8010 | Nginx 反代 `/video_feed/` 到视频流后端 |

```
推流：rtmp://{IP}:9090/stream/1
拉流：rtsp://127.0.0.1:8554/stream/1
观看：http://{IP}/video_feed/1
```

### 视频处理流水线

```
RTMP 帧 → 跳帧 → 缩放 480×480
    → 人脸检测识别 + 人数统计（dlib）
    → 行人检测 + 危险区域判断（HOG）
    → 异常检测：积水 / 着火 / 跌倒
    → 画框标注 → MJPEG 输出
    → 告警/事件 → POST Django /api/alerts/、/api/events/
```

### 6.2 核心业务场景

| 场景 | 触发条件 | 系统行为 | 告警类型 |
|------|----------|----------|----------|
| 家人识别 | 已注册家庭成员出现 | 绿框 + 显示姓名 | — |
| 陌生人 | 未注册人脸 | 红框 + 告警 | `FACE_UNKNOWN` |
| 人数统计 | 画面中多张人脸 | 更新在家人数/陌生人数 | 前端实时展示 |
| 小孩进厨房 | 角色=child 进入厨房禁区 | 告警 | `INTRUSION` |
| 积水 | 画面低处大面积蓝色/反光区域 | 告警 | `WATER` |
| 着火 | 高亮/红色火焰颜色区域 | 告警 | `FIRE` |
| 摔倒 | 人体框高宽比异常（躺倒） | 告警 | `FALL` |

## 10. 关键业务流程

### 10.1 人脸识别与人数统计

```
视频帧 → face 服务（AI 层）
    → 检测所有人脸 → 与家人员比对
    → 统计：total / family / stranger
    → 更新 GET /api/home/presence/（或 WebSocket 推送）
    → 陌生人 → POST /api/alerts/ (FACE_UNKNOWN)
    → MJPEG 标注框输出
```

### 10.2 危险区域（厨房禁小孩）

```
zone 配置：厨房多边形 + forbidden_roles=["child"]
    → detection 服务检测行人
    → 行人 + 人脸识别为 child + 进入多边形
    → POST /api/alerts/ (INTRUSION)
```

### 10.3 异常检测（积水/着火/跌倒）

```
每 N 帧执行：
    WATER  — 画面下方 HSV 蓝色/反光大面积 → POST /api/alerts/
    FIRE   — 高亮红/黄火焰颜色区域超阈值 → POST /api/alerts/
    FALL   — 人体框高/宽比低于阈值（躺倒）→ POST /api/alerts/
```
