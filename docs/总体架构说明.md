# 实时视频分析监测系统 — 总体架构说明

> **项目名称**：home-camera-monitor  
> **应用场景**：居家智能摄像头监控  
> **版本**：v1.1  
> **编制日期**：2026-07-06  
> **技术方案**：Vue3 + 全 Flask + Nginx-RTMP + MySQL

---

## 1. 文档目的

本文档描述居家摄像头监控系统的总体架构、技术选型、模块划分、数据流与部署方案，供立项评审、中期检查与结题验收使用。

---

## 2. 项目背景与目标

### 2.1 背景

现代家庭对居家安全与成员看护的需求日益增长。本系统基于家庭摄像头实时视频流，结合人脸识别与计算机视觉分析，实现对家庭成员的识别、在家人数统计、陌生人检测，以及厨房等危险区域的闯入告警，并对积水、着火、人员摔倒等异常情况进行自动识别与告警。

### 2.2 建设目标

| 目标 | 说明 |
|------|------|
| 家庭成员识别 | 注册家庭成员人脸，实时识别并标注身份 |
| 人员统计 | 统计当前画面/家中人数，区分熟人与陌生人 |
| 陌生人告警 | 未注册人脸出现即触发告警 |
| 危险区域 | 可配置禁区（如厨房禁止小孩进入），闯入即告警 |
| 异常检测 | 积水、着火、人员摔倒等紧急情况识别（≥2 种） |
| 告警管理 | 统一告警展示、处置、日志与事件回放 |

### 3.2 架构设计原则

| 原则 | 说明 |
|------|------|
| 单一后端 | 全 Python Flask，AI 与业务同进程 |
| 推拉流分离 | Nginx-RTMP 推流，Flask 拉流分析 |
| 场景驱动 | 功能围绕居家看护：识人、数人、禁区、异常 |
| 前后端分离 | 前端仅 HTTP 访问 API 与 MJPEG |

## 4. 技术选型

| 层次 | 技术 | 说明 |
|------|------|------|
| 前端 | Vue3 + Element Plus | 监控大屏、区域画框、告警列表 |
| 后端 | Flask + Blueprint | 视频 AI + 业务 API |
| 人脸识别 | dlib + face_recognition | 128 维编码，家人库比对 |
| 行人检测 | OpenCV HOG | 危险区域闯入判断 |
| 异常检测 | OpenCV 颜色/形态分析 | 积水、着火、跌倒 |
| 数据库 | MySQL | 用户、区域、告警、事件 |
| 流媒体 | Nginx-RTMP | 课程推拉流方案 A |

## 5. 推拉流方案

| 环节 | 协议 | 端口 | 说明 |
|------|------|------|------|
| 推流 | RTMP | 9090 | 摄像头/手机/OBS |
| 拉流 | OpenCV | — | Flask 内部 |
| 展示 | MJPEG | 5000 | 浏览器实时画面 |

```
推流：rtmp://{IP}:9090/live/living_room
观看：http://{IP}/video_feed/living_room
```

### 视频处理流水线

```
RTMP 帧 → 跳帧 → 缩放 480×480
    → 人脸检测识别 + 人数统计（dlib）
    → 行人检测 + 危险区域判断（HOG）
    → 异常检测：积水 / 着火 / 跌倒
    → 画框标注 → MJPEG 输出
```

### 6.2 核心业务场景

| 场景 | 触发条件 | 系统行为 | 告警类型 |
|------|----------|----------|----------|
| 家人识别 | 已注册家庭成员出现 | 绿框 + 显示姓名 | — |
| 陌生人 | 未注册人脸 | 红框 + 告警 | `FACE_UNKNOWN` |
| 人数统计 | 画面中多张人脸 | 更新在家人数/陌生人数 | 前端实时展示 |
| 小孩进厨房 | 角色=child 进入厨房禁区 | 告警 | `ZONE_INTRUSION` |
| 积水 | 画面低处大面积蓝色/反光区域 | 告警 | `FLOOD` |
| 着火 | 高亮/红色火焰颜色区域 | 告警 | `FIRE` |
| 摔倒 | 人体框高宽比异常（躺倒） | 告警 | `FALL` |

## 9. 数据库设计

| 字段 | 类型 | 说明 |

## 10. 关键业务流程

### 10.1 人脸识别与人数统计

```
视频帧 → face_service
    → 检测所有人脸 → 与家人员比对
    → 统计：total_count / family_count / stranger_count
    → 更新 /api/home/presence
    → 陌生人 → alert FACE_UNKNOWN
    → MJPEG 标注框输出
```

### 10.2 危险区域（厨房禁小孩）

```
zone 配置：厨房多边形 + forbidden_roles=child
    → detection_service 检测行人
    → 行人 + 人脸识别为 child + 进入多边形
    → alert ZONE_INTRUSION
```

### 10.3 异常检测（积水/着火/跌倒）

```
每 N 帧执行：
    FLOOD  — 画面下方 HSV 蓝色/反光大面积 → 告警
    FIRE   — 高亮红/黄火焰颜色区域超阈值 → 告警
    FALL   — 人体框高/宽比低于阈值（躺倒）→ 告警
```
