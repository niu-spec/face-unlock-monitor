# 实时视频分析监测系统 — 总体架构说明

> **项目名称**：face-unlock-monitor  
> **应用场景**：人脸识别开锁（智慧门禁）  
> **版本**：v1.0  
> **编制日期**：2026-07-06  
> **技术方案**：Vue3 + 全 Flask + Nginx-RTMP + MySQL

## 2. 项目背景与目标

### 2.1 背景

传统门禁依赖钥匙、密码或刷卡，存在丢失、泄露、代刷等问题。本系统基于实时视频流与人脸识别技术，实现「刷脸开锁」的智慧门禁方案，并辅以陌生人告警、门禁区域检测、尾随与徘徊检测，形成完整的出入口安防能力。

### 2.2 建设目标

| 目标 | 说明 |
|------|------|
| 刷脸开锁 | 已注册用户识别通过后自动开锁并记录通行日志 |
| 陌生人拦截 | 未注册人脸拒绝开锁并触发告警 |
| 区域安防 | 对门前限定区域进行闯入、过近、停留检测 |
| 异常检测 | 支持尾随进入、门口异常徘徊等至少 2 种异常行为告警 |
| 告警管理 | 统一告警展示、处置、日志与事件回放 |
| 可演示、可部署 | 支持云服务器部署，满足结题验收演示要求 |

## 3. 总体架构

### 3.1 架构概览

系统采用**前后端分离 + 单体 Flask 后端 + 独立流媒体服务**的分层架构。前端负责交互与展示；Flask 统一承担视频 AI 分析、业务 API 与数据持久化；Nginx-RTMP 负责接收多端视频推流。

```
┌─────────────────────────────────────────────────────────────────────┐
│                         展示层（Vue3 前端）                          │
│  门禁监控页 | 人脸录入 | 门锁状态 | 区域配置 | 告警中心 | 用户管理      │
└───────────────────────────────┬─────────────────────────────────────┘
                                │ HTTP / REST / MJPEG
┌───────────────────────────────▼─────────────────────────────────────┐
│                      应用层（Flask :5000）                            │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────────┐  │
│  │ video    │ │ face     │ │ detection│ │ business │ │ infra    │  │
│  │ 视频拉流  │ │ 人脸识别  │ │ 区域/异常 │ │ 用户/告警 │ │ 日志/回放 │  │
│  └──────────┘ └──────────┘ └──────────┘ └──────────┘ └──────────┘  │
└───────────────────────────────┬─────────────────────────────────────┘
                                │
              ┌─────────────────┼─────────────────┐
              │                 │                 │
              ▼                 ▼                 ▼
       ┌────────────┐   ┌────────────┐   ┌────────────┐
       │   MySQL    │   │ JSON/文件   │   │ Nginx-RTMP │
       │  业务数据   │   │ 人脸特征库  │   │  :9090     │
       └────────────┘   └────────────┘   └──────┬─────┘
                                                 │ RTMP 推流
                                          ┌──────▼──────┐
                                          │ 手机 / OBS  │
                                          │ 网络摄像头   │
                                          └─────────────┘
```

### 3.2 架构设计原则

| 原则 | 说明 |
|------|------|
| 单一后端 | 全 Python Flask，避免 Flask + Spring Boot 双后端联调成本 |
| 推拉流分离 | 推流由 Nginx-RTMP 负责，AI 分析由 Flask 拉流处理 |
| 模块内聚 | Blueprint 按业务域划分，AI 与 CRUD 同进程调用 |
| 前后端分离 | 前端仅通过 HTTP 访问 API 与 MJPEG 视频流 |
| 可演进 | 行人检测可从 HOG 升级为 YOLO；人脸库可从 JSON 迁入 MySQL |

## 5. 推拉流方案

### 5.1 方案说明

采用课程文档**推拉流方案 A**：RTMP 推流 + 服务端拉流 + Web 端 MJPEG 展示。

| 环节 | 协议/方式 | 端口 | 说明 |
|------|-----------|------|------|
| 推流 | RTMP | 9090 | 手机 EasyRTMP / PC OBS 推流 |
| 拉流 | RTMP → OpenCV | — | Flask 内部 `cv2.VideoCapture` |
| 展示 | MJPEG over HTTP | 5000 | 浏览器 `<img src="/video_feed/{id}">` |
| 录像（可选） | FLV 落盘 | — | Nginx `record_path` 配置 |

### 5.2 地址规范

```
推流：rtmp://{服务器IP}:9090/live/{stream_id}
拉流：rtmp://127.0.0.1:9090/live/{stream_id}    （Flask 内部）
观看：http://{服务器IP}/video_feed/{stream_id}   （经 Nginx 反代）
```

### 5.3 视频处理流水线

```
RTMP 帧输入
    → 跳帧（frame_skip=5，降低 CPU 负载）
    → 缩放（480×480）
    → 人脸检测与识别（dlib）
    → 行人检测与区域判断（HOG + 多边形）
    → 异常行为判定（尾随 / 徘徊）
    → 绘制标注框与文字
    → JPEG 编码（quality=70）
    → MJPEG 输出至前端
```

### 6.2 核心业务场景

| 场景 | 触发条件 | 系统行为 | 告警类型 |
|------|----------|----------|----------|
| 正常开锁 | 已注册用户刷脸，识别成功 | 绿框标注 → 下发开锁 → 记录通行日志 | — |
| 陌生人 | 人脸不在库中 | 红框标注 → 拒绝开锁 → 告警 | `FACE_UNKNOWN` |
| 区域闯入 | 未授权人员进入门禁区 | 告警，可选联动拒开 | `INTRUSION` |
| 距离过近 | 人员距区域边缘 < X px | 告警 | `PROXIMITY` |
| 异常停留 | 门前停留 > X 秒未认证 | 告警 | `LOITER` |
| 尾随进入 | 识别成功后短时间出现第二人 | 告警 | `TAILGATE` |

## 10. 关键业务流程

### 10.1 人脸识别开锁流程

```
┌────────┐    RTMP     ┌──────────┐    拉流     ┌─────────────┐
│ 摄像头  │ ─────────► │ Nginx    │ ─────────► │ Flask video │
└────────┘             └──────────┘            └──────┬──────┘
                                                      │
                                                      ▼
                                              ┌───────────────┐
                                              │ face_service  │
                                              │ 检测 → 编码比对 │
                                              └───────┬───────┘
                                                      │
                        ┌─────────────────────────────┼─────────────────────────────┐
                        │                             │                             │
                        ▼                             ▼                             ▼
                 ┌────────────┐              ┌────────────┐              ┌────────────┐
                 │ 识别成功    │              │ 陌生人      │              │ MJPEG 输出  │
                 │ door/unlock│              │ alert 入库  │              │ 前端展示    │
                 │ access_log │              │ 拒绝开锁    │              │ 标注框      │
                 └────────────┘              └────────────┘              └────────────┘
```

### 10.2 告警产生流程

```
视频帧 → detection_service
           ├─ 区域闯入 / 过近 / 停留
           ├─ 尾随（TAILGATE）
           └─ 徘徊（LOITER）
                ↓
         alert_service.create()
                ↓
         snapshot_service 保存截图
                ↓
         写入 MySQL alert 表
                ↓
         前端告警中心刷新展示
```

### 10.3 门锁状态（前端模拟）

本系统采用**软件模拟门锁**，不强制对接真实硬件：

| 状态 | 触发 | 前端展示 |
|------|------|----------|
| `locked` | 默认 / 开锁超时自动锁 | 红色锁图标 |
| `unlocked` | 识别成功 | 绿色开图标，N 秒后回锁 |
| `denied` | 陌生人 | 红色拒绝 + 闪烁告警 |

## 14. 性能与优化

| 策略 | 说明 |
|------|------|
| 跳帧 | `frame_skip=5`，每 5 帧处理 1 帧 |
| 分辨率 | 拉流后缩放至 480×480 |
| JPEG 质量 | 70，平衡清晰度与带宽 |
| 多 worker | gunicorn `-w 2`，视频流建议单 worker 或独立进程 |
| 检测算法 | 中期 HOG，后期可换 YOLOv8n |
