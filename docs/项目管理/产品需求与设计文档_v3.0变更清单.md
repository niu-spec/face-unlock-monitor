## 一、必改项（P0）

### 1.1 §2.3 技术方案表 — 整表重写

**现状问题**：仍写 Nginx-RTMP、HOG、MJPEG 主展示。

**改为**：

| 层次 | 技术选型 | 说明 |
|------|----------|------|
| 前端 | Vue3 + Vite + Element Plus | 监控页 WebRTC iframe + FaceOverlay Canvas；禁区编辑器 WebRTC 背景 |
| 业务后端 | Django 4.2 + DRF + SimpleJWT | 业务 API + Swagger；单体集成 AI |
| 人脸识别 | dlib + face_recognition | 128 维编码；多帧活体注册与反欺骗 |
| 目标检测 | YOLOv8n（HOG 降级） | 行人检测，配合禁区多边形 |
| 异常检测 | OpenCV HSV/形态 + PANNs 音频 | 积水/着火/摔倒 + 异常声学 + 音视频联动 |
| 数据库 | MySQL | 用户、家庭、区域、告警、事件、日报 |
| 流媒体 | OBS → RTMP(:9090) → MediaMTX → RTSP(:8554) | WebRTC(:8889) 主预览；MJPEG `/video_feed/` 后端备用 |
| AI 展示 | `/api/video/presence/` | 前端 200ms 轮询 overlay 框数据 |
| 部署 | gunicorn(:8010) + Nginx 反代 | 生产单体后端 |

**删除**：「Nginx-RTMP」「经 MJPEG 向前端输出带标注的实时画面」作为主方案表述。

### 1.3 §3.2.4 视频监控模块 — 业务流程重写

**删除/替换**：

- ❌ `前端通过 <img> 标签接收 MJPEG 流`
- ❌ `后端 video.py 通过 ffmpeg 从 Nginx-RTMP 拉取`
- ❌ UC-011 后置条件仅写「MJPEG 实时画面」

**改为**：

1. 管理员打开 `HomeMonitor.vue`，选择客厅/厨房摄像头  
2. 前端加载 **WebRTC iframe**（`http://{IP}:8889/stream/{id}/`）显示低延迟原始画面  
3. 前端每 **200ms** 轮询 `GET /api/video/presence/?stream_id={id}`，获取 `persons` / `face_boxes` / `alert_boxes`  
4. `FaceOverlay.vue` 在 Canvas 上叠加 AI 标注框  
5. `PersonStats.vue` 展示 total / family / stranger  
6. 停推流约 5s 后 presence 清空，显示「无推流信号」  
7. **MJPEG** `/video_feed/{id}` 保留为后端调试/备用入口（AI 标注已烧录在帧内，非监控页主路径）  

**用例表更新**：

| 用例 | 原后置条件 | 改为 |
|------|------------|------|
| UC-011 | MJPEG 实时画面 | WebRTC 预览 + Canvas AI 标注 |
| UC-013 | 输出 MJPEG 帧流 | RTSP 拉流 + process_frame + overlay API + MJPEG 备用 |
