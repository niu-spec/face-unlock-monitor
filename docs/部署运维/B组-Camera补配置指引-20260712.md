### 1.2 Camera 是干什么的

`Camera` 表把**业务视频流**和**家庭**绑在一起：

| stream_id（业务层） | 对应 OBS 推流 | 说明 |
|---------------------|---------------|------|
| `living_room` | `/stream/1` | 客厅 |
| `kitchen` | `/stream/2` | 厨房 |

AI 每帧处理时查询：

```text
stream_id=kitchen → Camera 表 → household_id=1
                              → 告警写入时带上 household_id=1
                              → 告警中心（家庭 1）可见
```

**没有 Camera 记录时**：检测仍执行，但告警 `household_id=null`，用户在告警中心看不到。

```text
有推流 + 有 Zone  →  能判定闯入/着火/摔倒
无 Camera          →  告警挂不到家庭  →  告警中心为空
```

## 3. 补配置（三选一，推荐方案 A）

### 方案 A：Django Shell 精确绑定（推荐）

**适用于**：线上已有用户家庭（如 ID=1），不要新建演示家庭。

将下面 `HOUSEHOLD_ID = 1` 改成上一步查到的实际 ID：

```bash
cd /service/home-camera-monitor/backend
python manage.py shell
```

```python
from apps.households.models import Household, Camera

HOUSEHOLD_ID = 1  # ← 改成实际家庭 ID

household = Household.objects.get(id=HOUSEHOLD_ID)

Camera.objects.get_or_create(
    household=household,
    stream_id="living_room",
    defaults={
        "name": "客厅摄像头",
        "rtsp_url": "rtsp://127.0.0.1:8554/stream/1",
        "is_active": True,
    },
)
Camera.objects.get_or_create(
    household=household,
    stream_id="kitchen",
    defaults={
        "name": "厨房摄像头",
        "rtsp_url": "rtsp://127.0.0.1:8554/stream/2",
        "is_active": True,
    },
)

print("done, cameras:", Camera.objects.filter(household=household).count())
exit()
```

## 5. 与禁区 / 异常检测的关系

| 能力 | 是否依赖 Camera |
|------|-----------------|
| YOLO 人体框、WebRTC 画面 | ❌ 不依赖 |
| 危险区域闯入判定 | ❌ 不依赖（依赖 Zone + 推流） |
| 着火 / 摔倒 / 积水判定 | ❌ 不依赖 |
| **告警写入家庭** | ✅ **依赖 Camera** |
| **告警中心列表展示** | ✅ **依赖 Camera + 登录家庭一致** |

因此：**Camera 未配 ≠ 检测坏了**，而是**告警看不到**。

## 6. 常见问题

**Q：补了 Camera 仍无告警？**

- 确认 `stream_id` 是 `living_room` / `kitchen`（不是 `1` / `2`）
- 确认登录家庭的 `X-Active-Household-Id` 与 Camera 的 `household_id` 一致
- 禁区测试：角色需在 `forbidden_roles` 内，或按陌生人逻辑触发；人脸未识别时按 `stranger` 处理
- 闯入需连续 ≥3 帧在禁区内

**Q：一个家庭可以只有厨房 Camera 吗？**

可以，但客厅流（`living_room`）的告警会缺少 `household_id`，建议两路都配。

**Q：需要改 Nginx 吗？**

不需要。Camera 是数据库配置，与 Nginx / MediaMTX 无关。
