## 2. 路径说明（本次问题的根源）

项目里存在 **两套前端路径**，文档历史上不一致：

| 路径 | 来源 | 说明 |
|------|------|------|
| `/service/home-camera-monitor/frontend/dist` | `deploy-all.sh` 的 `npm run build` 输出 | **当前推荐** |
| `/service/frontend/dist` | 旧版 `deploy-frontend.sh`、部分 nginx 配置 | 外网目前很可能仍读这里 |

`deploy/deploy-all.sh` **不会** 自动 rsync 到 `/service/frontend/dist`。  
若 nginx `root` 仍指向旧目录，跑多少次 `deploy-all.sh` 外网都不会变。

### 2.1 先确认 nginx 实际读哪个目录

```bash
# 宝塔常见路径
grep -R "root " /www/server/nginx/conf/ 2>/dev/null | grep -v "#"

# 或系统 nginx
grep -R "root " /etc/nginx/ 2>/dev/null | grep -v "#"
```

记下实际 `root` 路径，后续 build 产物必须落到该目录。

## 3. 推荐修复方案（二选一）

### 方案 A：改 nginx 指向仓库 dist（**长期推荐**）

一次改配置，以后只跑 `deploy-all.sh` 即可。

1. **宝塔面板**：网站 → 设置 → **网站目录**  
   改为：`/service/home-camera-monitor/frontend/dist`

2. **或手动改 nginx 配置**：

```nginx
root /service/home-camera-monitor/frontend/dist;
index index.html;

location / {
    try_files $uri $uri/ /index.html;
}
```

3. 执行 **第 4 节** 部署命令。

---

### 方案 B：保留 nginx 旧路径，手动同步 dist

适合暂时不能改 nginx 配置的情况。

```bash
cd /service/home-camera-monitor/frontend
npm ci
npm run build

# 创建目标目录（不存在时会报 rsync error）
sudo mkdir -p /service/frontend/dist

# 同步（注意末尾斜杠）
sudo rsync -av --delete /service/home-camera-monitor/frontend/dist/ /service/frontend/dist/

# 宝塔常见运行用户
sudo chown -R www:www /service/frontend/dist
```

**若无 rsync**，用 cp 代替：

```bash
sudo mkdir -p /service/frontend/dist
sudo rm -rf /service/frontend/dist/*
sudo cp -a /service/home-camera-monitor/frontend/dist/. /service/frontend/dist/
sudo chown -R www:www /service/frontend/dist
```

## 4. 标准部署步骤（请按顺序执行）

```bash
# ── 0. 进入项目 ──
cd /service/home-camera-monitor

# ── 1. 对齐最新 dev ──
git fetch origin dev
git status --short

# 若有本地改动导致 deploy-all 失败：
# git stash
# 或强制对齐（不删 .env.production 等未跟踪文件）：
git reset --hard origin/dev

git log -1 --oneline
# 期望：ef6c30b Merge pull request #32 ...

# ── 2. 环境检查 ──
test -f backend/.env.production && echo "env ok" || echo "缺少 backend/.env.production"
ffmpeg -version | head -1 || (sudo apt-get update && sudo apt-get install -y ffmpeg)
systemctl is-active home-camera-backend
docker ps | grep -i mediamtx

# ── 3. 一键部署（后端 + 前端 build + nginx reload）──
DEPLOY_BRANCH=dev bash deploy/deploy-all.sh

# ── 4. 若 nginx root 仍是 /service/frontend/dist，执行方案 B 同步 ──
# （见第 3 节方案 B，或改用方案 A 改 nginx）

# ── 5. 重启后端（可选，确保检测逻辑生效）──
sudo systemctl restart home-camera-backend

# ── 6. nginx 测试并重载 ──
sudo nginx -t && sudo systemctl reload nginx
# 宝塔 nginx 可能是：
# /www/server/nginx/sbin/nginx -t && /etc/init.d/nginx reload
```

### 日常仅需更新、不需重建 MediaMTX 时

```bash
DEPLOY_BRANCH=dev DEPLOY_MEDIAMTX=0 bash deploy/deploy-all.sh
```

### 需要重建 MediaMTX 容器时

```bash
DEPLOY_BRANCH=dev DEPLOY_MEDIAMTX=1 bash deploy/deploy-all.sh
```

## 5. 部署后验收（B 组在服务器执行）

### 5.1 版本与前端产物

```bash
git log -1 --oneline

# 以下二选一，取决于 nginx root
DIST=/service/home-camera-monitor/frontend/dist
# DIST=/service/frontend/dist

ls -lt $DIST/assets/HomeMonitor*.js | head -3
ls -lt $DIST/index.html

# 新前端必须能搜到异常框代码
grep -l "alert_boxes" $DIST/assets/*.js
grep -l "火情" $DIST/assets/*.js 2>/dev/null || grep -l "alert_boxes" $DIST/assets/*.js
```

**通过标准**：

- `HomeMonitor*.js` / `index.html` 修改时间为 **今天**
- `grep alert_boxes` 至少有一个 `.js` 文件命中
- 外网 `curl -s http://127.0.0.1/ | grep index-` 中的 bundle 名与 `$DIST/index.html` 一致

### 5.2 后端与 API

```bash
systemctl status home-camera-backend --no-pager | head -5

curl -s http://127.0.0.1:8010/api/video/status/ | python3 -m json.tool | head -30

curl -s "http://127.0.0.1:8010/api/video/presence?stream_id=2" | python3 -m json.tool | head -30

curl -s http://127.0.0.1:8010/api/detection/status/ | python3 -m json.tool

curl -s "http://127.0.0.1:8010/api/detection/audio/status/?stream_id=kitchen" | python3 -m json.tool
```

**通过标准**：

- `8010` 上 API 均返回 200
- 有人打开监控页且 OBS 推 stream/2 后，`audio/status` 中 `model_ready: true`，`kitchen` 的 `degraded: false`

### 5.3 Nginx 路由（必查）

```bash
curl -s -o /dev/null -w "frontend:%{http_code}\n" http://127.0.0.1/
curl -s -o /dev/null -w "swagger:%{http_code}\n" http://127.0.0.1/api/docs/
```

**`/api/` 与 `/video_feed/` 必须反代到 `127.0.0.1:8010`**，不能指到 8000。  
参考：[nginx/home-camera-monitor.conf.example](../../nginx/home-camera-monitor.conf.example)

### 5.4 流媒体

| 检查项 | 地址 | 通过标准 |
|--------|------|----------|
| OBS 推流（厨房） | `rtmp://152.136.29.158:9090/stream`，密钥 `2` | OBS 显示推流中 |
| WebRTC | `http://152.136.29.158:8889/stream/2/` | 有画面 |
| MJPEG | `http://152.136.29.158/video_feed/2` | 有画面或占位图 |

## 7. 常见问题

### 7.1 `deploy-all.sh` 报 `working tree has local changes`

```bash
git status
git stash
# 或
git fetch origin dev && git reset --hard origin/dev
```

### 7.2 `rsync` 报 `No such file or directory`

```bash
# 先 build
cd /service/home-camera-monitor/frontend && npm ci && npm run build

# 再建目录
sudo mkdir -p /service/frontend/dist

# 再同步
sudo rsync -av --delete dist/ /service/frontend/dist/
```

### 7.3 `rsync` 报 `Permission denied`

```bash
sudo rsync -av --delete /service/home-camera-monitor/frontend/dist/ /service/frontend/dist/
sudo chown -R www:www /service/frontend/dist
```

### 7.4 外网仍是旧页面

1. 确认 nginx `root` 与 `$DIST` 一致（第 2 节）
2. `sudo nginx -t && sudo systemctl reload nginx`
3. 浏览器无痕模式或强刷
4. 检查 `$DIST/index.html` 里 `index-*.js` 文件名是否已变化

### 7.5 有告警但没有彩色异常框

- 99% 是前端未更新到含 `alert_boxes` 的版本
- 按第 5.1 节 `grep alert_boxes` 排查

### 7.6 音频 `model_ready: false`

- 后端重启后模型懒加载，需 **打开监控页** 启动 Worker
- 确认 OBS **音频未关闭**（AAC）
- 确认 `ffmpeg` 已安装：`which ffmpeg`

### 7.7 `/api/video/presence` 404

- Nginx 未把 `/api/` 指到 **8010**
- 或后端未重启

## 8. 终极方案（前述均失败时）

保留生产密钥与数据，重新 clone 后再部署：

```bash
cp /service/home-camera-monitor/backend/.env.production /tmp/env.bak

cd /service
mv home-camera-monitor home-camera-monitor.bak.$(date +%Y%m%d)

git clone -b dev https://github.com/niu-spec/home-camera-monitor.git home-camera-monitor
cp /tmp/env.bak /service/home-camera-monitor/backend/.env.production

cd /service/home-camera-monitor
DEPLOY_BRANCH=dev bash deploy/deploy-all.sh

# 若 nginx 仍读 /service/frontend/dist：
sudo mkdir -p /service/frontend/dist
sudo rsync -av --delete frontend/dist/ /service/frontend/dist/
sudo chown -R www:www /service/frontend/dist

sudo systemctl restart home-camera-backend
sudo nginx -t && sudo systemctl reload nginx
```

> 不要删除 `home-camera-monitor.bak.*` 直到验收通过。  
> `mysql8-data/`、`yolov8n.pt` 等未跟踪文件见 [记录/生产目录文件策略.md](./记录/生产目录文件策略.md)。
