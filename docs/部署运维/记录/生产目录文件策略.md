# 生产目录文件策略

> **服务器**：152.136.29.158  
> **项目路径**：`/service/home-camera-monitor`  
> **依据**：B 组 CD 落地反馈（2026-07-10）  
> **维护**：A 组

---

## 原则

1. **Git 仓库只管理可版本化的代码**；生产密钥、模型权重、数据库数据 **不入库**。
2. Jenkins / 手动 Deploy **只更新 tracked 文件**，**不删除、不覆盖** 未跟踪的生产文件。
3. **默认不重启 MediaMTX**（`DEPLOY_MEDIAMTX=0`），避免打断推流。

---

## 未跟踪文件处理策略

| 路径 | 类型 | 策略 | 说明 |
|------|------|------|------|
| `backend/.env.production` | 生产环境变量 | **保留本地，不入 Git** | 已在 `.gitignore`；Deploy 前 `deploy-all.sh` 会 source 此文件 |
| `backend/yolov8n.pt` | YOLO 模型权重 | **保留本地，不入 Git** | 已在 `.gitignore`；首次部署需手动 `wget`/上传；或设 `YOLO_MODEL` 指向固定外部路径 |
| `mysql8-data/` | MySQL 数据目录 | **迁出 Git 工作树** | 建议移到 `/data/mysql8` 或 Docker volume；**禁止** `git clean -fdx` |
| `deploy/start_backend.sh` | systemd 启动脚本 | **纳入 Git** | 仓库已有 `deploy/start_backend.sh`；Deploy 用 `git checkout` 同步 tracked 版本即可 |

---

## Deploy 时的 Git 行为

### Jenkins CD（`Jenkinsfile`）

```text
git fetch → checkout dev → merge --ff-only $GIT_COMMIT
  ↓ 失败时
git checkout $GIT_COMMIT -- .   # 仅更新 tracked 文件
git update-ref refs/heads/dev $GIT_COMMIT
```

- **不会** `git reset --hard`（避免 B 组担心的工作区风险表述）
- **不会** 删除未跟踪文件（`.env.production`、`yolov8n.pt`、`mysql8-data/` 均保留）

### 手动 Deploy

```bash
cd /service/home-camera-monitor
git fetch origin dev
git merge --ff-only origin/dev   # 或 git pull --ff-only
DEPLOY_MEDIAMTX=0 bash deploy/deploy-all.sh
```

---

## MediaMTX 开关

| 场景 | 命令 |
|------|------|
| 日常 CD（默认） | `DEPLOY_MEDIAMTX=0 bash deploy/deploy-all.sh` → 日志：`[deploy] skip MediaMTX` |
| 重建/升级 MediaMTX | `DEPLOY_MEDIAMTX=1 bash deploy/deploy-all.sh` |

Jenkins Pipeline 环境变量已设 `DEPLOY_MEDIAMTX=0`。

## B 组 Deploy 前检查清单

```bash
cd /service/home-camera-monitor
git status --short          # 未跟踪文件应只有上表允许的项
test -f backend/.env.production && echo "env ok"
test -f backend/yolov8n.pt && echo "yolo ok" || echo "yolo missing (HOG fallback)"
systemctl is-active home-camera-backend
docker ps | grep home-mediamtx
```
