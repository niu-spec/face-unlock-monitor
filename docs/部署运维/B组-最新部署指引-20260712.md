## 1. 部署前检查

SSH 登录服务器后确认：

```bash
# 项目目录
ls /service/home-camera-monitor

# 后端环境（conda 或 venv 至少有一个）
conda env list | grep home-camera || ls /service/home-camera-monitor/backend/venv

# 生产配置存在
test -f /service/home-camera-monitor/backend/.env.production && echo "env ok"

# 服务状态
systemctl status home-camera-backend --no-pager
docker ps | grep -i mediamtx

# ffmpeg（新功能必需，没有就先装）
ffmpeg -version || (apt-get update && apt-get install -y ffmpeg)
```

### Nginx 路由（务必确认）

**`/api/` 与 `/video_feed/` 必须反代到视频/AI 后端 `127.0.0.1:8010`**，否则会出现「有画面但人数为 0、无 AI 框、presence 404」等问题。

```nginx
location /api/ {
    proxy_pass http://127.0.0.1:8010;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
}

location /video_feed/ {
    proxy_pass http://127.0.0.1:8010;
    proxy_buffering off;
    proxy_cache off;
    proxy_read_timeout 3600;
}
```

前端静态资源目录应指向构建产物，例如：

```nginx
root /service/home-camera-monitor/frontend/dist;
```

## 2. 一键部署（推荐）

```bash
cd /service/home-camera-monitor

# 若之前有本地改动，先清掉
git status
# 有未提交改动时：git stash  或  git reset --hard origin/dev

# 一键：拉代码 → migrate → 重启后端 → 构建前端 → reload nginx
DEPLOY_BRANCH=dev bash deploy/deploy-all.sh
```

脚本完成后确认版本：

```bash
git log -1 --oneline
# 期望看到 dec4aee 或更新，例如：
# dec4aee fix(test): balance clip settings lifecycle
```

如需同时重启 MediaMTX：

```bash
DEPLOY_BRANCH=dev DEPLOY_MEDIAMTX=1 bash deploy/deploy-all.sh
```

## 3. 手动分步部署（排错时用）

```bash
cd /service/home-camera-monitor
git fetch origin dev
git checkout dev
git pull --ff-only origin dev
git log -1 --oneline

# ── 后端 ──
cd backend
source venv/bin/activate    # 或 conda activate home-camera

pip install -r requirements.txt -q

# 数据库迁移（本次关键）
python manage.py migrate --noinput
# 涉及应用：accounts, alerts, events（clip_path / identity 字段）

python manage.py check
sudo systemctl restart home-camera-backend

# ── 前端 ──
cd ../frontend
npm ci
npm run build

# ── 流媒体（可选）──
bash ../deploy/mediamtx_run.sh

# ── Nginx ──
sudo nginx -t && sudo systemctl reload nginx
```

| 应用 | 迁移文件 | 内容 |
|------|----------|------|
| `accounts` | `0006_familymember_identity` | 家人 `identity` 字段 |
| `alerts` | `0005_alert_clip_path` | 告警 `clip_path` 字段 |
| `events` | `0003_event_clip_path` | 事件 `clip_path` 字段 |

## 4. 部署后验收

### 4.1 服务层

```bash
# 后端健康
systemctl status home-camera-backend --no-pager

# 视频 worker（应有 workers / presence 字段）
curl -s http://127.0.0.1:8010/api/video/status/ | python3 -m json.tool

# 外网首页
curl -s -o /dev/null -w "frontend:%{http_code}\n" http://127.0.0.1/

# Swagger
curl -s -o /dev/null -w "swagger:%{http_code}\n" http://127.0.0.1/api/docs/
```

### 4.2 流媒体

| 检查项 | 地址 | 通过标准 |
|--------|------|----------|
| OBS 推流 | `rtmp://152.136.29.158:9090/stream/1` | OBS 显示推流中 |
| WebRTC | `http://152.136.29.158:8889/stream/1/` | 有画面 |
| MJPEG | `http://152.136.29.158/video_feed/1` | 有画面或占位图 |

### 4.3 前端功能（A 组验收）

1. 登录系统 → 打开监控页，确认推流时有人脸框和人数统计
2. **停推流** → 约 5s 后人数应清零，页面显示「无推流信号」
3. 家人管理 → 「身份」列不再全是 `—`
4. 触发一条**新告警**（陌生人/闯入等）→ 等约 10s → 在告警中心/事件记录点「回放」→ 应能播放 MP4 短视频
5. 监控日报 → 「重点事件回放」表格有数据

> 注意：只有**部署后新产生的告警**才会有视频；旧告警只有截图。

## 5. 常见问题

### `deploy-all.sh` 报 working tree has local changes

```bash
git status
git stash
# 或强制对齐远程：
git fetch origin dev && git reset --hard origin/dev
```

### 家人身份仍显示 `—`

迁移未执行。手动跑：

```bash
cd /service/home-camera-monitor/backend
source venv/bin/activate   # 或 conda activate home-camera
python manage.py migrate accounts --noinput
sudo systemctl restart home-camera-backend
```

### 告警回放只有截图、没有视频

1. 确认 ffmpeg 已安装：`which ffmpeg`
2. 确认后端日志无录制错误：

```bash
journalctl -u home-camera-backend -n 100 --no-pager | grep -i clip
```

3. 确认 RTSP 有流（ffmpeg 从 `rtsp://127.0.0.1:8554/stream/{id}` 截取）

### 停推流后仍显示旧人脸

确认 `git log -1` 为最新 `dev`，且后端已重启。新逻辑：超过 5s 无新帧则清空 presence。

### `/api/video/presence/` 返回 404

Nginx 未把 `/api/` 指到 **8010**，或后端未更新到含该接口的版本。

## 6. 可选环境变量（`.env.production` 或 systemd）

默认值一般不用改，按需追加：

```bash
# 告警短视频（默认开启，录 10 秒）
EVENT_CLIP_ENABLED=true
EVENT_CLIP_SECONDS=10
FFMPEG_BIN=ffmpeg

# 推流过期阈值（秒）
PRESENCE_STALE_SECONDS=5
```

短视频存储目录：`backend/snapshots/clips/`（由 `CLIP_ROOT` 控制）。
