# B 组依赖修复重部署指引（2026-07-13）

> **发给**：B 苏哲勋（云服务器运维）  
> **协同验收**：A 牛雨昊 / D 李东礼（音频检测）  
> **服务器**：`152.136.29.158`  
> **仓库**：https://github.com/niu-spec/home-camera-monitor（分支 `dev`）  
> **背景**：PR #38（`201baf6`）部署时 `pip install` 因 `torch` / `torchaudio` 版本冲突失败，磁盘被占满，**服务未切换到新代码**  
> **预计耗时**：15～25 分钟

---

## 0. 一句话结论

**必须先 `git pull` 到含依赖修复的最新 dev，再 `pip install --no-cache-dir`，然后重启后端。**

| 组件 | 是否必做 | 原因 |
|------|----------|------|
| **git pull** | ✅ 必做 | 获取 `torchaudio` 移除 + PR #38 音频修复 |
| **pip install** | ✅ 必做 | 上次安装失败，旧进程仍在跑 |
| **后端 restart** | ✅ 必做 | 加载新代码与依赖 |
| **前端 rebuild** | ❌ 本次可跳过 | 本次仅修后端依赖，无前端变更 |
| **migrate** | ⚠️ 建议执行 | 无新 migration 也可跑，无害 |

## 1. 上次失败原因（请勿重复）

`requirements.txt` 中曾同时存在：

```text
torch>=2.2,<3.0
torchaudio~=2.0.0    ← 与 torch 2.2+ 不兼容
```

- `pip` 长时间回溯下载，**占满磁盘**
- B 组已清理 pip 缓存（约 5.7GB），当前可用空间约 **8.6GB**
- **虚拟环境未损坏**，但 **线上仍是旧后端进程**

**修复内容**：删除未使用的 `torchaudio`（音频检测代码只用 `torch`），保留 `torch>=2.2,<3.0`。

## 3. 部署步骤（按顺序执行）

### 3.1 拉取最新代码

```bash
cd /service/home-camera-monitor
git checkout dev
git pull --ff-only origin dev
git log -1 --oneline
```

### 3.2 确认 requirements.txt 已无 torchaudio

```bash
grep -E "torch|torchaudio" backend/requirements.txt
```

**期望输出**（有 `torch`，**无** `torchaudio`）：

```text
torch>=2.2,<3.0
```

### 3.3 安装依赖（关键：加 --no-cache-dir）

```bash
cd /service/home-camera-monitor/backend
conda activate home-camera

pip install --no-cache-dir -r requirements.txt
```

若 `torch` 下载较慢属正常（约 700MB+），**不要中断**。  
安装完成后：

```bash
pip check
python -c "import torch; print('torch', torch.__version__)"
```

### 3.4 数据库与 Django 检查

```bash
python manage.py migrate --noinput
python manage.py check
```

### 3.5 重启后端

```bash
sudo systemctl restart home-camera-backend
sudo systemctl status home-camera-backend --no-pager | head -8
```

### 3.6 重载 Nginx（可选，本次一般不需要）

```bash
sudo nginx -t && sudo systemctl reload nginx
```

## 4. 部署后验收

### 4.1 服务状态

```bash
systemctl is-active home-camera-backend
curl -s http://127.0.0.1/api/detection/status/
```

期望：`"status": "running"`，capabilities 含 `SCREAM`、`GLASS_BREAK`。

### 4.2 音频检测（需 A 组打开厨房监控页 + OBS 推流密钥 2）

```bash
curl -s "http://127.0.0.1/api/detection/audio/status/?stream_id=kitchen" | python -m json.tool
```

期望：

| 字段 | 期望 |
|------|------|
| `model_ready` | `true` |
| `running` | `true` |
| `degraded` | `false` |

### 4.3 类别标签映射（PR #38 重要修复）

```bash
sudo journalctl -u home-camera-backend --since "5 min ago" --no-pager | grep -iE "527 个 AudioSet|硬编码|类别索引"
```

**期望看到**：

```text
从模型元数据解析了 527 个 AudioSet 类别标签
```

若只看到 **「使用硬编码的 … 个 AudioSet 类别索引映射」**，请反馈 A/D 组（SCREAM 检测可能不准）。

### 4.4 音频诊断日志

A 组对着麦克风喊 6～8 秒时：

```bash
sudo journalctl -u home-camera-backend -f | grep "AUDIO DEBUG"
```

关注 `pcm_rms`（应有值）和 `target_probs` 中 `SCREAM` 是否 > 0.10。

### 4.5 RTSP 音频（可选）

OBS 推流中执行：

```bash
ffmpeg -rtsp_transport tcp -i rtsp://127.0.0.1:8554/stream/2 -vn -t 5 -af volumedetect -f null -
```

`mean_volume` 不应接近 `-91 dB`（那表示几乎静音）。

## 5. 常见问题

| 现象 | 处理 |
|------|------|
| `pip` 再次占满磁盘 | `pip cache purge` 后重跑 `pip install --no-cache-dir` |
| `torch` 安装超时 | 重试同一命令，勿改版本 |
| `No module named torch` | 确认 `conda activate home-camera` 后再 install |
| `model_ready: false` | 打开监控页触发懒加载；查 journalctl 是否有 PANNs 加载错误 |
| `degraded: true` | 确认 OBS 推流有音频；重启后端；重开监控页 |
| 服务 restart 失败 | `journalctl -u home-camera-backend -n 50 --no-pager` 贴 A 组 |
